## Work Completed

### Jetson Nano — Reflash

Original password was unknown. Entered recovery mode by shorting **FC_REC to GND**
on the board header near the camera connectors, then replugging USB.

Downloaded L4T R32.7.1 manually (no SDK Manager — version 2.4.0 dropped Jetson Nano
support):
- `Jetson-210_Linux_R32.7.1_aarch64.tbz2` (264 MB BSP)
- `Tegra_Linux_Sample-Root-Filesystem_R32.7.1_aarch64.tbz2` (1.4 GB rootfs)

Flash command that worked:
```bash
sudo ./flash.sh jetson-nano-devkit-emmc mmcblk0p1
```

> Note: `jetson-nano-emmc` and `mmcblk0boot0` are both invalid for this board.
> Correct combination is `jetson-nano-devkit-emmc` + `mmcblk0p1`.

First boot setup wizard choices:
- Network: skip (no Ethernet connected)
- Hostname: `lucia-jetson`
- User: `lucia` / `lucia-143-tomato`
- Nvpmodel: MAXN

### ZED SDK 3.8.2

Installed `zstd` dependency first, then downloaded SDK directly on Jetson:
```bash
wget https://download.stereolabs.com/zedsdk/3.8/l4t32.7/jetsons -O ZED_SDK_Jetson_JP46.run
./ZED_SDK_Jetson_JP46.run
```

CUDA was not in PATH despite being installed. Fixed:
```bash
sudo apt install cuda-toolkit-10-2 -y
echo 'export PATH=/usr/local/cuda/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc
```

Confirmed: `nvcc --version` → CUDA 10.2.300

SDK libraries confirmed at `/usr/local/zed/lib/` and `/usr/local/zed/tools/`.

Known issues from install:
- Python API failed (numpy pip build error on arm64) — non-critical
- `libv4l-dev` must NOT be installed — breaks hardware encode/decode


Confirmed ROS2 Humble works:

Configured NVIDIA container runtime in `/etc/docker/daemon.json`:
