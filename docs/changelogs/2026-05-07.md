### lucia_vision — Custom ZED ROS2 Node

Abandoned the official ZED ROS2 wrapper — all versions compatible with ROS2 Humble
require ZED SDK 4.x headers (`sl/Fusion.hpp`) which don't exist in SDK 3.8.2.

Wrote minimal custom C++ ROS2 node `src/ros2/lucia_vision/`:
- Publishes `/zed/rgb/image/compressed` (JPEG, ~14 Hz)
- Publishes `/zed/odom` (visual-inertial odometry, ~13 Hz)
- Publishes `/zed/objects` (3D object positions, ~13 Hz)

Build required several workarounds:
- Bypassed `zed-config.cmake` (requires BLAS which fails in container) — link ZED libs directly
- Mounted `libturbojpeg.so.0` from host into container at `/host_libs/`
- nlohmann-json installed from source (not in Ubuntu 18.04 apt)

**All three topics confirmed publishing. ZED 2i is live on ROS2.**

## How to Restart Next Session

### 1. SSH into Jetson
```bash
ssh lucia@192.168.55.1   # via USB
# password: lucia-143-tomato
```

### 2. Make sure ZED 2i is plugged into blue USB 3.0 port and barrel jack power is connected

### 3. Pull latest repo
```bash
cd ~/repos/group-project-group-37-project-l-u-c-i-a
git pull
```

### 4. Launch the Docker container
```bash
docker run -it --rm \
  --runtime nvidia \
  --network host \
  --privileged \
  -e ROS_DOMAIN_ID=42 \
  -e RMW_IMPLEMENTATION=rmw_fastrtps_cpp \
  -e PATH=/usr/local/cuda/bin:$PATH \
  -e LD_LIBRARY_PATH=/usr/local/zed/lib:/host_libs:/usr/local/cuda/lib64:$LD_LIBRARY_PATH \
  -v /usr/local/zed:/usr/local/zed \
  -v /usr/local/cuda:/usr/local/cuda \
  -v /usr/lib/aarch64-linux-gnu/libturbojpeg.so.0:/host_libs/libturbojpeg.so.0 \
  -v ~/repos/group-project-group-37-project-l-u-c-i-a/src/ros2:/ros2_ws/src \
  dustynv/ros:humble-ros-base-l4t-r32.7.1 \
  bash
```

### 5. Build (only needed if code changed) and run
```bash
cd /ros2_ws
colcon build --packages-select lucia_vision --cmake-args=-DCMAKE_BUILD_TYPE=Release
source install/setup.bash
ros2 run lucia_vision zed_publisher
```

### 6. Verify topics (second terminal)
```bash
ssh lucia@192.168.55.1
docker exec -it $(docker ps -q) bash
source /ros2_ws/install/setup.bash
ros2 topic list
```
