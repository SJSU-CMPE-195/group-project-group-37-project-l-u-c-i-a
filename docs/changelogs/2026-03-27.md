### `update.sh` (new file, repo root)

Created automated repo update script for the Pi. The Pi uses `wlan0` exclusively as a WiFi access point (NetworkManager `pi-ap` profile + dnsmasq), so it cannot simultaneously connect to another network for internet access.

The script automates the full cycle:

1. Stops dnsmasq and brings down the `pi-ap` AP connection.
2. Stops the system `wpa_supplicant` service and kills any stale instances.
3. Removes any stale `/run/wpa_supplicant/wlan0` socket.
4. Starts a temporary `wpa_supplicant` instance to connect to the `lucia` WiFi network.
5. Obtains an IP address via `dhcpcd`.
6. Runs `git fetch --all && git clean -fd && git reset --hard @{u}` to force-pull the latest code (discards local changes).
7. Kills the temporary `wpa_supplicant`, restores the `pi-ap` AP, and restarts dnsmasq.
8. A `trap restore EXIT` ensures the AP is always restored even if the script fails.

Key debugging notes from this session:
- The system `wpa_supplicant` service (PID 800) was conflicting with the temporary instance — fixed by stopping it before starting our own.
- `dhclient` is not available on Raspberry Pi OS; `dhcpcd` is used instead.
- `sudo mkdir -p /run/wpa_supplicant` created the directory with wrong permissions and broke socket creation — reverted; socket is created automatically by `wpa_supplicant` itself.
