## Architecture Decisions

### Sensor roles

| Sensor | Board | Role |
|--------|-------|------|
| RPLidar A2 (A2M8) | Raspberry Pi | 2D SLAM, localization, static obstacle mapping |
| Stereolabs ZED 2i | Jetson Nano | 3D depth, labeled object detection, dynamic obstacles |

The two sensors are **complementary, not redundant.** The RPLidar is 2D — it cannot
identify what it detects or see above/below its scan plane. The ZED 2i fills that gap
with labeled 3D detections (people, furniture, etc.).

### Integration layer

**ROS2 Jazzy** selected as the communication and integration layer over ZMQ, MQTT,
and plain TCP. Rationale:

- Stereolabs provides an official `zed-ros2-wrapper`
- `rplidar_ros2` + `slam_toolbox` is the standard 2D SLAM path
- Nav2 ties everything together (path planning + dynamic costmap from ZED detections)
- ROS2 DDS handles inter-board topic routing automatically over direct Ethernet — no
  broker or custom socket code needed

### ROS2 runs in Docker on the Pi

The Pi runs Debian 13 (Trixie). ROS2 Jazzy targets Ubuntu 24.04 (Noble) — its apt
packages depend on Ubuntu-specific library names that do not exist in Trixie and
cannot be installed directly. Docker with the official `ros:jazzy-ros-base` arm64
image is the solution. USB devices are passed through via `--device` flags and
`--network=host` keeps ROS2 DDS discovery working across the network.

### Inter-board communication

Direct **Gigabit Ethernet cable** between Pi and Jetson Nano (no router).
- Pi static IP: `192.168.1.1`
- Jetson static IP: `192.168.1.2`
- `ROS_DOMAIN_ID=42` on both boards

### Jetson Nano power

- Barrel jack (5.5mm/2.1mm center-positive), 5V/4A minimum
- **J48 jumper must be shorted** to enable barrel jack mode
- Powered by a dedicated 5V/4A power bank, separate from the Pi's X1202 UPS
- Pi and Jetson must be on **separate power rails** to avoid inrush current at Jetson
  boot affecting the Pi

## Hardware Notes

- **RPLidar A2 USB:** Connects to Pi via included USB-UART adapter at 115200 baud.
  Both the Roomba and RPLidar use CP210x chips — differentiate by `serial` attribute
  in udev rule.
- **ZED 2i USB:** Must use the blue USB 3.0 port on the Jetson Nano.
- **Jetson Nano:** Not yet sourced. All Jetson-side work is blocked on hardware.
